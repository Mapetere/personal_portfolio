LatterPay

A production grade WhatsApp bot built to manage donations and payments with end to end security, resilience, and a deep focus on providing a reliable service for those who give.

Python 3.9+ Flask WhatsApp API Docker Terraform

The Mission

LatterPay was born from a simple but powerful realization: technology should be a quiet bridge, not a stumbling block, between a generous heart and a community in need. I built this WhatsApp integration to ensure that the act of giving is as seamless and heartfelt as a conversation with a friend. It's about transforming a complex payment flow into a moment of simple connection.

Technical Vision

Beyond the interface, LatterPay is a study in resilient state management. In regions where network stability can falter, I designed a robust system that treats every donation with the highest level of care. By implementing idempotent processing and custom transaction trackers, I ensured that no contribution is ever lost—protecting the legacy of every donor's intent.

Key Features

WhatsApp Integration

Full WhatsApp Business API integration via Pygwan. Supports text messages, interactive buttons, and list menus.

Payment Processing

Paynow integration supporting EcoCash, OneMoney, TeleCash, and USD payments with real time status updates.

End to End Encryption

Meta Flow encryption with AES 256 CBC and RSA for secure data exchange, protecting every contribution with the highest standards.

Circuit Breaker

Prevents cascading failures when external services are down. Auto-recovers after 60 seconds.

Rate Limiting

Token bucket algorithm protects against abuse with 30 requests per user limit.

Analytics Dashboard

An admin panel with report generation, payment statistics, and daily summaries to keep the organization informed and empowered.

Deployment

LatterPay can be deployed using Docker, Railway, Heroku, or Terraform:

# Docker
docker build -t latterpay .
docker run -p 8010:8010 --env-file .env latterpay

# Railway
railway login && railway deploy

# Terraform
cd terraform && terraform init && terraform apply
